PROBLEMA REAL ATUAL:
Quando um usuário:

* desliga Wi-Fi
* perde internet abruptamente
* troca de rede
* entra em túnel/elevador
* mobile suspende rede

o websocket do Supabase demora alguns segundos para morrer oficialmente.

Resultado:

* o timer continua rodando
* o sistema não detecta disconnect imediatamente
* depois ocorre reconnect normalmente

Além disso:
NÃO QUEREMOS falsos positivos.
Micro instabilidades NÃO devem:

* pausar sessão
* gerar reconnect spam
* gerar loops de pause/resume
* causar flickering online/offline

OBJETIVO:
Implementar uma arquitetura de:

* heartbeat watchdog
* soft disconnect
* grace period
* anti-thrashing
* reconnect cooldown

==================================================
2. SOFT DISCONNECT
==================

Criar 3 estados distintos:

CONNECTED
SUSPECTED_DISCONNECT
DISCONNECTED

Fluxo correto:

0–8 segundos sem heartbeat:

* continuar normal

8–15 segundos sem heartbeat:

* entrar em SUSPECTED_DISCONNECT
* NÃO pausar timer ainda
* exibir “Reconectando parceiro…”
* NÃO broadcast pause
* NÃO alterar timer

15+ segundos sem heartbeat:

* confirmar disconnect
* pausar timer automaticamente
* persistir timer_remaining_seconds
* broadcast TIMER_PAUSE
* status “paused_disconnect”

IMPORTANTE:
Micro perdas NÃO devem congelar a sessão.

==================================================
3. ANTI-THRASHING
=================

Adicionar proteção contra loops.

Após:

* pause automático
* reconnect automático
* resume automático

Criar cooldown mínimo:

* 10 segundos

Durante cooldown:

* ignorar novas mudanças automáticas de estado
* evitar pause/resume em loop

==================================================
4. RECONNECT ESTÁVEL
====================

Quando heartbeat voltar:

SE estava em:

* SUSPECTED_DISCONNECT:
  apenas restaurar CONNECTED
  SEM pausar/resumir sessão

SE estava em:

* DISCONNECTED:
  executar reconnect real:

  * resumeSession()
  * novo timer_started_at
  * re-anchor
  * broadcast TIMER_RESUME
